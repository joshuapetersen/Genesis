## THE THREE-GATE VERIFICATION
Before any command with potential system-wide impact, hardware modification, or data risk is proposed or executed, the following assessment must be presented to the Architect:

1. **TOTAL FAILURE COST (TFC):**
   - Explicitly define the worst-case scenario. 
   - What happens if the code is 100% wrong? (e.g., Data loss, OS corruption, Hardware lockout).

2. **REVERSION PATH (RP):**
   - Is there a "Step Back"? 
   - Define exactly how to return to the current state (e.g., Backup location, Undo command, Manual reset).

3. **"STATIC" RISK (SR):**
   - Identify potential triggers for system security overrides (e.g., BitLocker, TPM, Secure Boot, BIOS Lockouts).

- **LEVEL 0 (PURE)**: Read-only, diagnostic. No risk.
- **LEVEL 1 (VOID)**: Modifies external media (USB/VHD) or temporary files. Low risk.
- **LEVEL 2 (DYNAMIC)**: Modifies system settings, installs packages, or moves non-critical files. Moderate risk.
- **LEVEL 3 (ANALYTIC)**: Modifies partitions, bootloaders, or critical data. High TFC. Mandatory Architect Sign-off.
- **LEVEL 4 (CRITICAL)**: BIOS changes, hardware flashing, or partition wiping. Maximum TFC.
